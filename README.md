# Executive Intelligence Briefing (EIB)

> Every solution should leave behind a better starting point for the next person.

---

## Why EIB Exists

Every day, people solve problems that have already been solved.

Projects are repeated.

Processes are reinvented.

Lessons learned disappear when people retire, change jobs, or simply forget why a decision was made.

The Executive Intelligence Briefing (EIB) was created to change that.

The goal is not merely to collect information.

The goal is to transform experience into reusable knowledge.

---

## What Is EIB?

EIB is a structured knowledge system.

It combines documentation, templates, prompts, playbooks, reports, and personal knowledge into a single version-controlled repository.

The result is a living system that improves over time.

---

## Repository Strategy

This project consists of two complementary repositories.

### Public Repository

Reusable knowledge.

Templates.

Documentation.

Examples.

Prompts.

Guides.

No personal information.

### Private Repository

Personal knowledge.

Health.

Financial planning.

Retirement.

Projects.

Reports.

Private implementations of Public templates.

---

## Guiding Principle

> Every solution should leave behind a better starting point for the next person.

---

## Engineering Principles

- Document the reasoning, not just the decision.
- Templates before personalization.
- Public when practical. Private when necessary.
- Simplicity over complexity.
- Version everything.
- Improve continuously.

---

## Vision

Create a framework that anyone can adopt.

Provide enough structure that people spend less time reinventing systems and more time solving meaningful problems.
