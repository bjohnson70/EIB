# AI Backlog

## Purpose

The backlog captures work that has been identified but is not currently the highest priority.

Unlike `NEXT_TASK.md`, which always contains a single active objective, this document maintains the broader list of future work. As priorities change, items may move between this backlog and `NEXT_TASK.md`.

The backlog should be reviewed regularly and reordered as the project evolves.

---

# Priority Levels

Priority is assigned using the following scale:

| Priority | Description |
|----------|-------------|
| P1 | Critical - Current milestone blocker |
| P2 | High - Needed soon |
| P3 | Medium - Valuable enhancement |
| P4 | Low - Future improvement |
| P5 | Someday / Nice to Have |

# Backlog Management Rules

When reviewing this document:

1. Remove completed work.
2. Reorder priorities as needed.
3. Move the single highest-priority task to `AI/NEXT_TASK.md`.
4. Record significant changes in `AI/SESSION_NOTES.md` if appropriate.
5. Archive obsolete items rather than deleting them when historical context is valuable.

---

# Guiding Principle

The backlog is intentionally aspirational.

Not every idea needs to be implemented. Its purpose is to capture opportunities so they are not forgotten while keeping the current focus clear.
